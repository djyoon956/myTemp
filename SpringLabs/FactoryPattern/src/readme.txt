팩토리 메서드
	객체지향디자인 패턴
	Factory 클래스를 이용하여 객체 생성
	
	
	장점
		객체의 추상화 단계가 높아진다
		객체 생성이 팩토리 메소드에 캡슐화가 이루어진다.
		사용자는 생성하는 객체에 대해 알 필요가 없어진다.
		객체 추가 삭제가 용이해진다
		객체의 자료형이 하위 클래스에 의해 결정된다.
		구상 클래스에 의존하지 않고 추상화된 것에 의존한다.
		
	단점
		객체가 추가 되거나 삭제 될때마다 코드를 변경해야한다.

		
		팩토리 패턴은 템플릿 패턴을 활용한다.
		템플릿 패턴 : 전체적인 흐름을 제공하는 template method와 추사 메서드를 하위 클래스가 구현하여 흐름을 만들어간다.
						template method에서 추상 메서드를 호출한다.
